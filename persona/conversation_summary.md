# Conversation Summary

## هدف
به جای نگه داشتن هزاران پیام خام، مکالمات طولانی را خلاصه کن.

## ساختار User Profile

### فیلدهای پروفایل
```
User Profile
├── سبک صحبت: [رسمی / خودمونی / فنی / شوخ]
├── موضوعات علاقه: [تکنولوژی، تاریخ، شوخی، ...]
├── سطح دانش: [مبتدی / متوسط / حرفه‌ای]
├── شوخی مورد علاقه: [کتلت، نان و کتلت، ...]
├── لحن ترجیحی: [جدی / شوخ / دوستانه]
└── نکات مهم گفتگو:
    ├─ [تاریخ] درباره X صحبت کرد
    ├─ [تاریخ] سوال Y پرسید که بلد نبودم
    └─ [تاریخ] از پاسخ Z خوشش آمد
```

### قوانین خلاصه‌سازی
- هر ۱۰ پیام → یک خلاصه جدید
- خلاصه قبلی + خلاصه جدید = پروفایل به‌روز
- نکات مهم (تاریخ، علاقه، رویداد) نگه داشته شود
- نکات تکراری حذف شود

## نحوه استفاده
- پروفایل کاربر به جای تاریخچه کامل به مدل داده شود
- مدل می‌داند این کاربر کیست بدون اینکه ۵۰ پیام قبلی را بخواند
- حافظه طولانی‌مدت کاربر در user_memory ذخیره شود
