# PROMPT — ساختار مستندات پروژه

## REPORT/ — وضعیت و تحلیل پروژه

```
REPORT/
├── REPORT.md        # وضعیت کلی پروژه، تکنولوژی‌ها، قابلیت‌ها
├── STRUCTURE.md     # ساختار کامل فایل‌ها و پوشه‌ها (درخت)
├── BOT_STRUCTURE.md # معماری بات، جریان پیام، دیاگرام
├── CHANGES.md       # تاریخچه تغییرات با تاریخ و هش کامیت
├── INFO.md          # مشخصات فنی، متغیرهای محیطی، محدودیت‌ها
├── DESIGN.md        # تصمیمات معماری، فلسفه طراحی
├── SECURITY.md      # مسائل امنیتی، آسیب‌پذیری‌ها، توصیه‌ها
└── PROMPT.md        # این فایل — توضیح ساختار مستندات
```

## SECRETS/ — تنظیمات و راهنما

```
SECRETS/
├── 01-bot-token.txt           # توکن ربات از BotFather
├── 02-admin-ids.txt            # آیدی عددی ادمین‌ها
├── 03-api-worker.txt           # کلید API (Groq و ...)
├── 04-master-password.txt      # رمز پیش‌فرض پنل مدیریت
├── 05-default-settings.txt     # تنظیمات پیش‌فرض (مدل، آستانه‌ها)
├── botfather-commands.txt      # لیست دستورات برای BotFather
├── README.txt                  # معرفی سریع و خلاصه
└── SETUP_GUIDE.md              # راهنمای گام‌به‌گام نصب
```

## قوانین ساخت

### REPORT/
- هر فایل باید **self-contained** باشه (نیاز به خوندن فایل دیگه نداشته باشه)
- تاریخ‌ها به فرمت YYYY-MM-DD
- از Markdown استفاده کن
- سکشن‌ها با `##` جدا بشن

### SECRETS/
- فایل‌های ۰۱ تا ۰۵: هر کدوم فقط یه خط مقدار + توضیح کوتاه
- فایل‌های متنی (`botfather-commands.txt`, `README.txt`): اطلاعات کاربردی و مستقیم
- `SETUP_GUIDE.md`: مراحل قدم به قدم با جزئیات
- هیچوقت توکن واقعی رو توی فایل‌های متنی نذار (فقط placeholder)
